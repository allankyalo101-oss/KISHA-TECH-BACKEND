The Premise

Most hardware shops in Kenya run on WhatsApp messages, exercise books, and memory. K-Tech is not opening as a hardware shop. It is opening as a technology company that sells electronics. The difference is not cosmetic — it is structural.

The same infrastructure already powering CloudOven — Supabase, Vercel, Render, Twilio, and the Zovrix AI employees — can be leveraged for K-Tech at near-zero marginal cost. Sarah, who already closes sales and handles customer support for CloudOven, can be extended to work the K-Tech WhatsApp line.

The goal of this document is not to impress. It is to give the founder and the investor a clear, honest picture of what we are building, what it costs, what it unlocks, and in what order we do things.

"We do not mix. We move with longevity attitude and calm strategic approach."

— Founder directive, April 2026
🏗
Separation of Concerns
K-Tech gets its own isolated Supabase project, its own Vercel deployment, its own Render service. CloudOven and K-Tech share infrastructure providers — not infrastructure itself.
📐
MVP First, Scale Always
Week 1: the shop opens with a working POS and inventory system. Nothing more. Every additional feature is added only when the business operation demands it.
🤖
Automate What Repeats
Every task a human does more than 3 times a day is a candidate for automation. Restock alerts, daily summaries, customer receipts, WhatsApp replies — zero human effort, zero cost.
💸
Free Until It Hurts
We extract maximum value from every free tier before spending a shilling. When we pay, it is because the revenue justifies it — not because it was convenient.

The Stack

The architecture has six layers. Each layer has a clear owner, a clear tool, and a clear cost. Nothing is speculative.

Layer 1 Volta OS — Shop Management App Free · Browser
The single-file HTML application (already built). Runs entirely in the browser — no server needed for core shop operations. Works offline. The admin's daily tool: POS, inventory, transactions, reports, restock planner, customer ledger. In Phase 1, it connects to Supabase for cloud sync and becomes a proper PWA installable on any Android phone.
POSInventoryTransactions ReportsRestock PlannerCustomer Ledger Groq AI (Ask + Insights)Daily Briefing WhatsApp ReceiptsPrintable ReportsOffline PWA
Layer 2 Vercel — Frontend Hosting Free Tier · Hobby
K-Tech gets its own Vercel project (separate from CloudOven). Hosts the Volta OS app as a public URL and optionally a customer-facing product catalogue. Vercel's edge network means it loads fast everywhere in Kenya, including on mobile data. Custom domain e.g. app.ktechke.com when ready.
ktech-os.vercel.app (now) app.ktechke.com (Phase 2) Customer Catalogue CI/CD via GitHub
Layer 3 Render — Backend API Free Tier · Web Service
A lightweight Node.js / Express service — K-Tech's own instance, separate from CloudOven's backend. Handles: Twilio WhatsApp webhooks, M-Pesa Daraja callbacks, Groq API calls server-side (key stays server-side, never in the browser in production), and scheduled jobs (daily summary email, weekly backup trigger). Free tier on Render is sufficient for Phase 1 traffic.
WhatsApp WebhookM-Pesa Callback Groq Server-Side ProxyScheduled Jobs (node-cron) Receipt GeneratorInventory API
Layer 4 Supabase — Database + Auth + Realtime Free Tier · 500MB · 50k rows
K-Tech's own Supabase project (not CloudOven's). Stores all transactions, inventory, customers, sales, restock history. Supabase Realtime means if the owner is on the laptop and the staff is on the phone, both see live updates. Row-level security ensures staff can only write — not read financials. Free tier supports years of a single shop's data volume.
transactionsinventorycustomers salesrestock_logdaily_snapshots Realtime syncRow Level Security Auth (PIN / email)Storage (receipts)
Layer 5 Groq — AI Engine (Llama 3.3 70B) Free Tier · 14,400 req/day
Replaces Anthropic API for all AI features. Groq's free tier is extremely generous — 14,400 requests per day at zero cost. Llama 3.3 70B is strong enough for every financial analysis, restock recommendation, daily briefing, and conversational query the shop will generate. Key stored server-side on Render (not in browser) in production.
Ask AI (conversational)Daily Auto-Briefing AI InsightsSmart Price Suggestions Auto Transaction CategorisationRestock Intelligence WhatsApp Bot Brain
Layer 6 Twilio + Zovrix — Communications Strategic · Shared Infra
Here is where K-Tech inherits power from CloudOven at zero additional cost. The same Twilio account already running Sarah for CloudOven can host a second WhatsApp number for K-Tech. Sarah's framework (Zovrix AI) can be configured with K-Tech's personality, inventory knowledge, and business rules. When a customer WhatsApps the shop: Sarah replies, checks stock, sends prices, takes orders, and hands off to a human only when needed. This is enterprise-grade automation at startup cost.

Infrastructure Map

This shows how K-Tech and CloudOven share providers without mixing systems.

Supabase Account
cloudoven-db ≠ ktech-db
Two separate projects on the same Supabase account.
Completely isolated databases, auth systems, RLS rules.
No data crosses between them. Ever.
Vercel Account
cloudoven.vercel.app ≠ ktech-os.vercel.app
Two separate Vercel deployments, two GitHub repos.
Same account, same billing, zero interference.
Each has own env variables and build settings.
Render Account
cloudoven-api ≠ ktech-api
Two separate Web Services on Render.
K-Tech backend handles: WhatsApp webhooks, Daraja callbacks, Groq proxy, cron jobs.
Free tier: 750 hrs/month each.
Twilio Account
CloudOven number + K-Tech number
Same Twilio account, different WhatsApp numbers.
Each number has its own webhook URL (pointing to respective Render services).
Sarah's brain (Zovrix) is configured per-instance.
⚡ Key Decision

The Groq API key lives on the Render backend — not in the browser. In the MVP (offline HTML file), the key is in the browser temporarily. The moment Supabase + Render are connected, the key moves server-side. This is non-negotiable for production security.

What We Build Next

The MVP Volta OS file needs these changes before the shop opens next week:

🔄
1. Switch to Groq
Replace all Anthropic API calls with Groq's OpenAI-compatible endpoint. Model: llama-3.3-70b-versatile. Same intelligence, 14,400 free calls/day.
🌅
2. Auto Daily Briefing
On dashboard load, if no briefing today: pull yesterday's data, send to Groq, render a 3-line morning brief. Cached in localStorage so it only fires once per day.
📈
3. Fast Movers Panel
Dashboard shows top 5 selling items (by units and by revenue), bottom 5 dead stock items. Updates every time the dashboard loads.
💬
4. WhatsApp Receipt
After POS checkout, receipt modal has a "Send via WhatsApp" button. Opens wa.me with pre-filled receipt text. Customer receives on their phone instantly.
🏷
5. Smart Pricing
In Add Item form: type buying price, Groq suggests a selling price based on your shop's average margin. One click to accept.
🖨
6. End-of-Day Report
One button in dashboard: generates a formatted daily closing summary — sales, items sold, expenses, cash to take to bank. Print-ready.

These six additions complete the MVP. Everything else is Phase 1 and beyond. The rule is: the shop opens first, the tech grows with it.
